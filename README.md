# Ясно — концепт-редизайн: лендинг + воронка подбора психолога

Прототип для демонстрации трёх growth-гипотез. Не является официальным сайтом
Ясно: все данные — mock, аналитика — заглушка с логом в консоль.

## Запуск

```bash
npm install
npm run dev      # http://localhost:5173
npm run build    # production-сборка без ошибок TypeScript
```

Стек: Vite · React 18 · TypeScript (strict) · Tailwind CSS v4 · motion · lucide-react.
Без UI-китов — весь визуал кастомный. Бренд «Ясное небо 2.0»: белый воздух, фирменный голубой #279BE0 (CTA на #1873B5 — AA с белым), Geologica (display) + Inter (body) + Caveat (рукописные завитки-подписи в духе Ясно). Шрифты самохостятся (fontsource), фото психологов — локальные ассеты, квиз/дыхание/QA — lazy-чанки (лендинг ~80 КБ gzip).

**Ночная тема «мягкое ночное небо»**: глубокий сине-серый #0F151E (не чёрный) включается автоматически ночью (22–5, как Adaptive Dawn) и при системной тёмной теме; переключатели — луна в шапке и «Панель заботы». Класс `night` ставится inline-скриптом до первого пейнта — ночному посетителю не вспыхивает белый экран. Все поверхности сидят на токене `paper`, свет (сферы дыхания, солнце, текст CTA) остаётся белым. QA: `?hour=23` / `?hour=13` (форс главнее системной темы).

## Система «Станет ясно»

Сквозная механика бренда: посетитель приходит в тумане — уходит яснее.
Переменная `--clarity` (0 → 1, см. `lib/clarity.ts` + `.fog-veil`/`.clarity-reveal`
в `index.css`) управляет туманом, размытием и насыщенностью:

- **Adaptive Dawn** (`lib/dawn.ts`, `components/DawnBackdrop.tsx`) — палитра
  hero-фона следует за временем суток (5 фаз: предрассветная → утренняя →
  дневная → закатная → ночная), кроссфейд 4 с; ночью hero добавляет строку
  «Не спится? Мы рядом — и в 4 утра». QA: `?hour=4|8|13|19|23`.
- **«Расчистка»** (`src/clearing/`) — тесты как отдающий разговор: одно
  отражение на каждый ответ, туман рассеивается по ходу (`clarityForProgress`),
  прогресс — облачка→солнца, слайдер интенсивности, PNG-карточка результата
  (canvas, полностью на клиенте), бесплатный инструмент в каждом результате,
  ветка поддержки с телефонами помощи на крайних ответах. Плашка приватности:
  ответы не покидают браузер (проверено: ноль сетевых запросов).
- **«Момент тишины»** (`src/breath/BreathScene.tsx`) — полноэкранное дыхание
  4-4-6: кольца, генеративные «споры», звуковые ориентиры (Web Audio, off по
  умолчанию), пауза при скрытой вкладке, выход Esc/тапом.
- **Подбор, который цитирует** (`quoteFromAnswers` в `lib/matching.ts`) —
  карточки результатов связывают выбор с ответами пользователя («Вы отметили
  тревогу — Анна работает именно с этим») и проявляются из тумана.
- **Панель заботы** (`src/care/`) — плавающая кнопка слева внизу: тихий режим
  (kill-switch движения/звука/вибрации), контраст, крупный текст, меньше
  анимации, звук. Состояние живёт в React (ничего не пишем на устройство), `useCalmMotion()` объединяет
  системный `prefers-reduced-motion` и настройки панели.

QA-страница со всеми состояниями: `http://localhost:5173/qa`.

## Три гипотезы — как проверить кликами

### 1. Сепарация выбора и оплаты — ценность до регистрации
Квиз → подборка → слот → и только потом регистрация (email, телефон — по желанию).

Путь: «Подобрать психолога» (или чип эмоции в hero — тогда старт сразу
с шага тем с выбранным чипом) → 6 шагов с живым счётчиком «подходят N»
→ анимация подбора, цитирующая ответы → 3 карточки с честным «Совпадение N%»
(45–97, объяснение по клику «Как считаем?») и блоком «Почему именно она/он»
→ «Выбрать время» (слоты из единого источника `lib/slots.ts` — совпадают
с карточками) → **SignupGate** «Почти готово» → «Запись закреплена».
С карточки психолога «Записаться» ведёт сразу в слот-пикер.

### 2. Тесты как acquisition-воронка
Секция «Мини-тесты» (#tests): «Уровень тревоги» (GAD-2) · «Выгорание» ·
«Подойдёт ли мне терапия?». Результат сразу, без email-гейта, с дисклеймером
«не является диагнозом». CTA результата открывает квиз с уже выбранной темой
(шаг 2 предзаполнен) — как и чипы эмоций в hero.

### 3. Активация второй сессии как LTV-рычаг
Экран подтверждения: «что будет на первой сессии», «Добавить в календарь»
(генерация .ics на лету), и мягкий офер «закрепить ритм» с опорой на «81%
чувствуют результат к 5-й сессии». Принятие — тост «Слот предварительно
удержан на 24 часа»; отказ — «Решу после первой сессии», без давления.

| 3 | Три шага с мини-мокапами, нарисованными кодом | `HowItWorks` |
| 4 | «Расчистка»: тесты-разговор — ДО каталога (гипотеза 2) | `ClearingSection` |
| 5 | Психологи: фильтры из единого источника тем, «Проверено Ясно», честные слоты | `TherapistsSection`, `TherapistCard` |
| 6 | Отбор 9% + Цены — единый «акт доверия» на mist-полотне, цитата у цены, правило 12 часов | `SelectionFunnel`, `Pricing` |

| 9 | FAQ: порядок по страху, выход в поддержку | `FAQ` |
| 10 | Финальный CTA + футер без мёртвых ссылок | `FinalCTA`, `Footer` |

`steps/*` → `MatchingAnimation` → `Results` → `SlotPicker` → `SignupGate`
→ `Confirmation`.

## Аналитика

`lib/analytics.ts` — `track(event, props)` → `console.info('[track]', …)`.
Инструментирована вся воронка: `cta_click`, `chip_click`, `quiz_start/step/back/abandon/complete`,
`match_shown`, `match_explain_open`, `therapist_expand`, `slot_selected`,
`signup_shown/submitted`, `booking_confirmed`, `second_session_offer_shown/accepted`,
`test_start/complete`, `test_card_saved`, `test_support_branch`,
`breath_start/complete`, `email_capture`, `night_toggle`, `care_toggle`.
Откройте консоль и пройдите воронку.

## Качество

- `npm run build` — без ошибок и предупреждений TypeScript (strict).
- Адаптив от 360px: грид психологов 3→2→1, квиз и sticky CTA проверены на 375px.
- Доступность: один h1, семантика, alt у фото, видимый фокус (`--sky`),
  квиз управляется с клавиатуры (стрелки/Enter/Esc), голубой CTA #1873B5 с белым текстом (5.0:1).
- `prefers-reduced-motion`: декоративные анимации отключаются, дыхательная
  практика переходит на текстовый таймер.
- Никаких тёмных паттернов: без фейковых таймеров и «осталось 2 места» —
  только честная доступность слотов.
